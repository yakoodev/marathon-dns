# DNS CLI (Marathon / Steam fix presets)

Простой интерактивный **PowerShell-скрипт** для Windows 10/11, который в один клик переключает DNS на заранее заданные пресеты (в т.ч. те, что часто помогают с ошибками подключения/региональными проблемами в играх и сервисах), умеет **менять DNS местами (swap)**, **восстанавливать исходные настройки** и отдельно **включать автоматический DNS (DHCP)**.

> Скрипт сам запросит права администратора (UAC) и после переключения выполнит очистку DNS-кэша.

---

## Что умеет

- ✅ Меню “как в запрете”: нажми `1/2/3/...` → Enter
- ✅ Пресеты DNS:
  - **Preset A:** `31.192.108.180` (основной) + `176.99.11.77` (резервный)
  - **Preset B:** `80.78.247.254` (основной) + `176.99.11.77` (резервный)
- ✅ **SWAP**: поменять местами primary/secondary (иногда помогает)
- ✅ **Restore Original DNS**: вернуть именно те DNS, которые были сохранены до первого изменения
- ✅ **Set Automatic DNS (DHCP)**: принудительно включить автоматическое получение DNS
- ✅ `ipconfig /flushdns` + (по возможности) `Clear-DnsClientCache`

---

## Как это работает (простыми словами)

Некоторые игры/сервисы могут работать нестабильно из-за того, **какие IP-адреса возвращает DNS** (GeoDNS, CDN, маршрутизация у провайдера, фильтрация и т.п.).  
Смена DNS может привести к тому, что домены начнут резолвиться в **другие рабочие узлы/маршруты**, и проблема пропадёт — без VPN.

⚠️ Важно: скрипт меняет **только DNS**, он **не меняет внешний IP**.

## Установка

1) Скачай `marathon-dns.ps1` (или склонируй репозиторий)
2) Открой PowerShell и перейди в папку со скриптом:
   ```powershell
   cd path\to\repo
   ```
3) Если скрипты запрещены политикой (делается один раз):
   ```powershell
   Set-ExecutionPolicy -Scope CurrentUser RemoteSigned
   ```
4) Запуск:
   ```powershell
   .\marathon-dns.ps1
   ```

Скрипт сам запросит права администратора.

## Использование

После запуска увидишь меню:

- `1` — включить **Preset A** (31.192.108.180 + 176.99.11.77)
- `2` — включить **Preset B** (80.78.247.254 + 176.99.11.77)
- `3` — **SWAP**: поменять местами текущие DNS (например, поставить `176.99.11.77` первым)
- `4` — **Restore Original DNS**: восстановить сохранённые исходные DNS
- `5` — **Set Automatic DNS (DHCP)**: включить автоматическое получение DNS для активных IPv4 интерфейсов
- `6` — **Статус**
- `0` — выход

---

## Чем отличаются Restore Original DNS и Set Automatic DNS (DHCP)?

- **Restore Original DNS** возвращает именно то состояние, которое скрипт сохранил перед первым изменением DNS:
  - если раньше был `AUTO/DHCP`, вернётся `AUTO/DHCP`
  - если раньше были ручные DNS, вернутся именно они
- **Set Automatic DNS (DHCP)** ничего не восстанавливает из state-файла, а просто включает автоматическое получение DNS на активных интерфейсах.

Если у тебя роутер раздаёт DNS по DHCP как `192.168.10.1`, то после **Set Automatic DNS (DHCP)** или восстановления исходного режима `AUTO/DHCP` Windows может показывать именно этот адрес.  
Это нормально: значит DNS берётся автоматически у роутера.

---

## Где хранится “как было”

При первом изменении DNS через **Preset A**, **Preset B** или **SWAP** скрипт сохраняет текущие настройки DNS в файл:

- `C:\ProgramData\dns-cli-state.json`

После **Restore Original DNS** (пункт `4`) файл удаляется.

## Советы по проверке результата

- После смены DNS лучше перезапустить проблемное приложение/игру/Steam.
- Проверить DNS можно в меню скрипта (`6`) или так:
  ```powershell
  Get-DnsClientServerAddress -AddressFamily IPv4
  ```

---

## Предупреждения

- Скрипт меняет DNS для **активных IPv4 интерфейсов**, которые сейчас в состоянии `Connected`.
- Если у тебя подняты виртуальные адаптеры/туннели (VirtualBox/Hyper-V/Radmin и т.п.), они тоже могут попасть в список активных.
  - Если хочешь ограничить только `Ethernet`/`Wi-Fi`, просто сделай issue/PR или поправь фильтр в `Get-ActiveIPv4Ifaces`.
